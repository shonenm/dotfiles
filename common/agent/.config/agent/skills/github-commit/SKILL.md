# Commit - Git コミット作成

現在のセッションで行った変更を論理的な単位に分割し、適切なコミットメッセージでコミットを作成します。

## 引数
| 引数 | 説明 |
|------|------|
| (なし) | セッション内の変更のみをコミット（デフォルト） |
| `all` | すべての未コミット変更をコミット |
| `<path>...` | 指定ファイル/ディレクトリのみを対象 |

## コミットメッセージ規約
- フォーマット: `prefix(scope): 説明`（英語、小文字始まり）
- prefix: `feat`, `imprv`, `fix`, `rfac`, `perf`, `docs`, `chore`

## 注意事項
- `--amend` はユーザーが明示的に要求しない限り使用しない
- 機密ファイル（`.env`等）はコミットしない
- 変更が単一の論理単位なら1コミットでよい
- `git push` は明示的に要求されない限り実行しない
